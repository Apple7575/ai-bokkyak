첨부한 이미지를 디자인 레퍼런스로 참고하여, 고령층 사용자를 위한 AI 음성 복약 관리 모바일 앱 UI를 만들어 주세요.

앱 이름은 **모두의 복약**입니다.

이 앱의 핵심 사용자는 스마트폰 사용이 익숙하지 않은 고령층입니다.
복잡한 기능을 많이 보여 주기보다, 큰 글씨와 명확한 버튼을 사용하여 쉽게 복약 일정을 등록하고, 약 먹을 시간이 되면 음성 안내를 받고, “먹었어요”라고 말하거나 버튼을 눌러 기록을 남길 수 있어야 합니다.

---

## 1. 디자인 방향

첨부한 레퍼런스 이미지와 유사한 분위기로 디자인해 주세요.

### 전체 분위기

* 신뢰감을 주는 의료·헬스케어 서비스
* 흰색 배경 중심의 깔끔한 화면
* 블루 계열을 메인 컬러로 사용
* 너무 차갑거나 병원 시스템처럼 딱딱하지 않게 구성
* 고령층도 쉽게 이해할 수 있는 단순한 정보 구조
* 카드 UI와 둥근 모서리를 적절히 사용
* 과도한 그라데이션, 화려한 애니메이션, 장식적인 요소는 최소화
* 한 화면에 너무 많은 정보를 넣지 말 것
* 아이콘과 텍스트를 함께 사용하여 기능을 직관적으로 표현

### 컬러 스타일

아래 컬러를 기준으로 디자인 토큰을 만들어 주세요.

* Primary Navy: `#102A5E`
* Primary Blue: `#2563EB`
* Secondary Blue: `#4F8EF7`
* Light Blue Background: `#EEF5FF`
* Card Background: `#FFFFFF`
* Border: `#D8E5F6`
* Main Text: `#1F2937`
* Secondary Text: `#64748B`
* Success Green: `#36B37E`
* Warning Orange: `#F5A623`
* Danger Red: `#E25353`
* Condition Purple: `#8B5CF6`

### 타이포그래피

* 한글 폰트는 `Pretendard` 또는 `Noto Sans KR`
* 본문 글씨는 최소 16px
* 핵심 안내 문구는 20~24px
* 화면 제목은 22~26px
* 숫자와 복약 시간은 크고 선명하게 표시
* 너무 얇은 글씨는 사용하지 말 것
* 고령층이 읽기 쉽게 줄 간격을 충분히 확보

### 접근성 기준

* 주요 버튼 높이는 최소 56px
* 터치 영역은 최소 48px 이상
* 화면마다 가장 중요한 행동 버튼은 1개 또는 2개만 강조
* 버튼에는 아이콘과 텍스트를 함께 표시
* 상태를 색상만으로 구분하지 말고 텍스트와 아이콘도 함께 표시
* 고령층이 실수해도 쉽게 이전 단계로 돌아갈 수 있게 구성
* 복잡한 키보드 입력보다 음성 입력과 버튼 선택을 우선 사용

## 3. 제작할 화면 목록

총 12개의 모바일 화면을 만들어 주세요.

---

### 화면 1. 스플래시 화면

#### 목적

앱 실행 시 서비스의 핵심 가치를 간단하게 보여 줍니다.

#### 구성

* 앱 로고
* 앱 이름: `모두의 복약`
* 문구: `말로 쉽게 기록하는 복약 관리`
* 하단 로딩 표시

#### 디자인

* 흰색 배경
* 중앙에 블루 계열 로고
* 심플하고 편안한 분위기

---

### 화면 2. 온보딩 화면

#### 목적

사용자에게 앱의 핵심 기능을 짧게 소개합니다.

#### 구성

3장의 카드 또는 슬라이드로 구성해 주세요.

1. `약 먹을 시간이 되면 음성으로 알려드려요`
2. `“먹었어요”라고 말하면 자동으로 기록돼요`
3. `보호자도 복약 여부를 함께 확인할 수 있어요`

하단에는 큰 버튼을 배치해 주세요.

* `시작하기`

#### 디자인

* 글씨를 크게 표시
* 카드마다 단순한 일러스트 아이콘 사용
* 설명은 한 문장 이내로 제한

---

### 화면 3. 홈 화면

#### 목적

사용자가 오늘의 복약 일정과 다음 알림 시간을 바로 확인합니다.

#### 상단 구성

* 인사말: `안녕하세요, 김영희 님`
* 보조 문구: `오늘도 건강한 하루 보내세요.`
* 알림 아이콘
* 보호자 연결 상태 아이콘

#### 핵심 카드

* 제목: `다음 복약 시간`
* 시간: `오늘 오후 6:30`
* 약 이름: `저녁 약`
* 강조 버튼: `복약 일정 보기`

#### 오늘의 복약 일정

카드 목록으로 표시해 주세요.

* 아침 약 · 오전 8:30 · `복용 완료`
* 점심 약 · 오후 12:30 · `복용 완료`
* 저녁 약 · 오후 6:30 · `복용 예정`
* 취침 약 · 오후 9:30 · `복용 예정`

#### 하단 요약

* 제목: `오늘의 복약 현황`
* 원형 Progress: `50%`
* 문구: `오늘 4번 중 2번 복용했어요`

#### 디자인

* 가장 중요한 다음 복약 시간을 가장 크게 표시
* 복약 완료는 초록색 체크 아이콘
* 복용 예정은 블루 계열 시계 아이콘
* 미복용은 빨간색 느낌표 아이콘

---

### 화면 4. 복약 관리 목록 화면

#### 목적

등록된 약과 복약 일정을 확인합니다.

#### 상단

* 제목: `복약 관리`
* 필터 탭:

  * 전체
  * 복용 중
  * 보관 중
  * 종료

#### 약 카드 예시

* `고혈압약`

  * 1일 1회 · 아침
  * 상태: `복용 중`
* `점심 약`

  * 1일 1회 · 점심
  * 상태: `복용 중`
* `비타민D`

  * 1일 1회 · 점심
  * 상태: `복용 중`
* `종합비타민`

  * 1일 1회 · 아침
  * 상태: `복용 중`

#### 하단 고정 버튼

* `+ 약 등록하기`

---

### 화면 5. 약 등록 방식 선택 화면

#### 목적

고령층 사용자가 편한 방식으로 약을 등록합니다.

#### 제목

`어떻게 약을 등록할까요?`

#### 선택 카드

1. `음성으로 간편 등록`

   * 설명: `말씀해 주시면 자동으로 입력해 드려요`
   * 가장 강조하여 표시
2. `버튼으로 직접 등록`

   * 설명: `항목을 하나씩 눌러 등록할 수 있어요`
3. `사진으로 등록`

   * 설명: `약 봉투나 약 포장을 촬영해 주세요`
   * 오른쪽 상단에 `준비 중` Badge 표시

#### 디자인

* 음성 등록 카드를 Primary Blue로 강조
* 각 카드에 큰 아이콘 사용
* 선택지를 너무 작게 만들지 말 것

---

### 화면 6. 음성으로 약 등록 화면

#### 목적

사용자가 말로 약 이름과 복약 시간을 등록합니다.

#### 제목

`음성으로 약 등록`

#### 중앙 구성

* 큰 원형 마이크 버튼
* 주변에 간단한 음성 파형
* 안내 문구:

  * `복용하는 약과 시간을 말씀해 주세요`
  * 예시: `매일 아침 8시에 고혈압약을 먹어요`

#### 음성 인식 후 표시

* 인식된 문장:

  * `매일 아침 8시에 고혈압약을 먹어요`
* 자동 분류 결과:

  * 약 이름: `고혈압약`
  * 복용 주기: `매일`
  * 복용 시간: `오전 8:00`

#### 하단 버튼

* Primary: `이대로 등록하기`
* Secondary: `다시 말하기`

---

### 화면 7. 버튼으로 약 등록 화면

#### 목적

음성이 익숙하지 않은 사용자도 버튼을 눌러 복약 일정을 설정합니다.

#### 입력 항목

* 약 이름

  * 직접 입력
  * 예시 Placeholder: `예: 고혈압약`
* 복용 시간대

  * `아침`
  * `점심`
  * `저녁`
  * `취침 전`
* 세부 시간

  * `7시`
  * `8시`
  * `9시`
  * `직접 설정`
* 반복 요일

  * `매일`
  * 월, 화, 수, 목, 금, 토, 일

#### 하단 고정 버튼

* `저장하기`

#### 디자인

* 입력창을 많이 만들지 말고 Chip과 큰 버튼 중심으로 구성
* 선택된 버튼은 Primary Blue 배경으로 강조

---

### 화면 8. AI 음성 복약 알림 화면

#### 목적

약 먹을 시간이 되었을 때 사용자가 바로 알아차릴 수 있도록 음성 안내를 제공합니다.

#### 전체 화면 Modal 형태

* 중앙 상단에 큰 벨 아이콘
* 음성 파형 애니메이션 느낌
* 시간: `오후 6:30`
* 제목: `저녁 약 드실 시간이에요`
* 안내 문구:

  * `김영희 님, 저녁 약을 복용할 시간입니다.`
  * `복용하신 뒤 말씀해 주세요.`

#### 중앙 마이크 버튼

* 큰 원형 버튼
* 텍스트: `말하기`

#### 하단 버튼

* `복용했어요`
* `아직 안 먹었어요`
* `30분 뒤 다시 알려주세요`

#### 디자인

* 시니어 사용자가 한눈에 알아볼 수 있도록 문구와 버튼을 크게 표시
* 화면에서 가장 중요한 행동은 `말하기` 버튼

---

### 화면 9. STT 음성 응답 처리 화면

#### 목적

사용자의 짧은 음성 응답을 인식하고 상태를 기록합니다.

#### 예시 상태 1

* 사용자 음성: `먹었어요`
* 인식 결과: `복용 완료`
* 성공 아이콘: 큰 초록색 체크
* 안내 문구: `복약 완료로 기록했어요`

#### 예시 상태 2

* 사용자 음성: `30분 뒤에 알려줘`
* 인식 결과: `재알림 요청`
* 시계 아이콘
* 안내 문구: `30분 뒤에 다시 알려드릴게요`

#### 예시 상태 3

* 음성 인식 실패
* 경고 아이콘
* 안내 문구: `잘 듣지 못했어요. 다시 말씀해 주세요.`
* 버튼:

  * `다시 말하기`
  * `버튼으로 선택하기`

각 상태를 Variant로 만들어 주세요.

### 화면 10. 복약 후 상태 확인 화면

#### 목적

#### 질문 3

`오늘 컨디션은 어떤가요?`

선택 버튼:

* `좋음`
* `보통`
* `나쁨`

#### 하단 버튼

* `기록 완료`

#### 디자인

* 한 화면에서 질문을 순서대로 보여 주되, 각 질문의 선택지는 단순하게 구성
